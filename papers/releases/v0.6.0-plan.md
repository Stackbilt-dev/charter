# v0.6.0 Release Plan

## Goal

Stabilise core CLI workflows, then ship targeted ADF architecture improvements that
reduce coupling and unlock safer extension for future releases.

## Non-Goals

- Major ADF parser redesign (full rewrite deferred to v0.7+)
- Breaking CLI command renames
- Premature contributor documentation for module boundaries still in flux

## Prioritized Feedback Inputs

- [ADX-003](../AGENT_DX_FEEDBACK_003.md): install/setup automation friction
- [ADX-005](../AGENT_DX_FEEDBACK_005.md): UX and output ergonomics findings
- [ADX-006](../AGENT_DX_FEEDBACK_006.md): docs/planning preset request
- [ADX-007](../AGENT_DX_FEEDBACK_007.md): ADF architecture cohesion and enforcement boundaries

## Focus Items (Curated)

- Harden install and setup guidance for non-interactive and sandboxed flows
- Reduce noisy JSON payload defaults in migrate-like workflows
- Improve first-run error remediation hints for common platform path and permission failures
- Document recommended command path transition (`npx` bootstrap -> pinned local `pnpm exec`)
- Split bundler into focused modules and unify enforcement pipeline (ADX-007)

## Version Bump Gate

**Do not bump to 0.6.0 until:**

1. Phase 1 and Phase 2 are merged and CI is green
2. `doctor --adf-only --ci` and `adf evidence --auto-measure --ci` contracts pass
3. Manual smoke test of `npx @stackbilt/cli bootstrap` on a clean project
4. Phase 3 is at minimum in-flight (P0 refactors landed or PR-ready)

Phase 4 items may ship in 0.6.x patches or defer to 0.7.0.

## Acceptance Criteria

- Automation path documented with one primary happy-path flow and one fallback flow
- At least one high-friction ADX-005 finding is shipped with clear before/after behavior
- No regression in `doctor --adf-only --ci` and `adf evidence --auto-measure --ci` contracts
- Bundler split (#8) and evidence pipeline (#9) landed with focused unit tests
- Bootstrap bugs (#4, #5) verified fixed via integration test or manual smoke
