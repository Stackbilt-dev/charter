# v0.16.0

Minor release adding ADF write-back tooling — closing the LM→ADF automation loop.

## What's new

### `updateEvidence` MCP tool

`charter serve` now exposes `updateEvidence`: an agent calls it after making code changes and Charter automatically measures the affected source files, diffs the values against what's stored in ADF, and patches the metrics in place.

```json
{ "tool": "updateEvidence" }                              // full refresh
{ "tool": "updateEvidence", "dryRun": true }              // preview only
{ "tool": "updateEvidence", "metrics": ["total_loc"] }    // specific key
```

Returns `{ measured, changes, skipped, written, constraints }` — agents see exactly what changed and whether constraints pass after the update.

### `adf patch` before/after diff

`charter adf patch --format json` now returns a `changes[]` array alongside `opsApplied`. Each entry records what was in the document before the patch and what it became after, so callers can audit exactly what changed rather than inferring from a count.

## Fixed

- **`charter serve` no longer hangs on startup errors** ([#157](https://github.com/Stackbilt-dev/charter/issues/157)): The MCP SDK's `stdin` handle was being acquired at module load time, keeping the Node process alive past any thrown `CLIError`. Both SDK modules are now lazily imported inside `serveCommand()` after path guards pass. A bad `--ai-dir` exits in under 100 ms with a structured JSON-RPC error and non-zero exit code.

- **`charter stamp-policies` structured error + env fallback**: `CHARTER_POLICY_REPO_REF` env var accepted as fallback when `--policy-repo-ref` is not passed. Failure emits a structured JSON error with `code: POLICY_REPO_REF_UNRESOLVED` and a `recoveryCommand` hint.
